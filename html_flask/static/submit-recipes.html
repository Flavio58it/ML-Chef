About the Final Project


DU Data Viz Final Project (Machine Learning) - Required Project Questions
1. Team Name / Team Project: Team Foodie/ML Chef
2. Problem Description: Team Foodie set out to answer the age old question, "What's for dinner?" Oftentimes, we look in our fridges, scanning for something to eat and don't know what to do with the ingredients we have available. With ML Chef, end users could take a picture of 3 food items in their fridge or in the store, have a machine learning model accurately classify those food items and have the database return recipes that utilize those ingredients.
3. Prediction methods used: regression, classification, multi-label classification
4. Two or more required libraries: Python Pandas, Python MatPlotLib, HTML/CSS/Bootstrap, MongoDB
5, 6, 7. Our score function was image classification and was based on accuracy and loss. This was both plotted and visually represented with a confusion matrix.
8. Our best accuracy score was 65%.
9. With our use of several different models, we had mixed results. Some provided highly accurate scores, some provided largely inaccurate scores.
10. Our training vs. testing scores suggested that the models were trained appropriately, neither over- nor under fitting.
11. Models used for classification: VGG16, VGG19, Xception, Inception V3, Transfer Learning and a custom CNN created specifically for this project.
12. These models and project were run locally and not deployed to an external environment.
13. We created a UI for user input and model prediction.
14. We didn't use a tool that was exclusively a visualization tool.
15. We did do textual analysis.
16. We didn't use any simple statistics.
17, 18. We used the following steps:
30% Data Collection (i.e. prepping images, finding data online)
5% Data Cleaning/Transformation/"PreProcessing" (i.e. pd merge)
5% Data Exploration (i.e. Jupyter Notebook and Excel)
5% Data Visualization (i.e. matplotlib, classification report, cluster, 'label' and 'probability')
3% Data Test/Train Split (i.e. cross validation)
15% Model Training (i.e. fiting)
10% Comparing Multiple Models (i.e. VGG16, VGG19, Custom CNN, Transfer Learning, Xception, Inception v.3)
5% Tuning your Individual Models
20 % Building a UI to Interact and WebScraping (i.e. CoLab, HTML, CSS, bootstrap)
2% Compiling/Running (i.e. running Flask locally)
19. We did not "tune your hyperparameters" with a technique like GridSearchCV.
20. We used standard train-test splits for image classification.
21. We used Jupyter notebook and CoLab for some of our data cleaning and exploration.
22. We used scatterplot, confusion matrix and correlation matrix.
23. We struggled with crazy results from the model, problems with connectivity and utilizing a variety of tools and data types to work together
24 and 25. Question not applicable to image classification
26. The number of classes that we were trying to predict varied by model.
About

Team Foodie was established in July 2019, through the University of Denver's Data Analytics Bootcamp."What 's for Dinner?", also known as ML-Chef, is the third and final project of their bootcamp experience. Their collaborative experience utilizes web scraping, a python flask app, MongoDB, machine learning models and various statistical summaries.
